# Bat_OS Wave 3 — FS · ED · CM design source

JSX artifacts for the interactive-app trio (Files, Editor, Comms).
Generated by Claude Design (April 2026) in the same session as
the lock-screen, desktop-shell, and Wave 2 apps.

## Files

- **`apps-fs-ed-cm.jsx`** — three pane components:
  - `FSPane` — encrypted-vault data table with selection state.
  - `EDPane` — file-tab bar + gutter + syntax-colored code +
    status strip.
  - `CMPane` — connection pill header + chat timeline + composer
    with char counter.
- **`wave3-specs.jsx`** — implementation spec sheet covering
  shared strips, FS table column widths + selection rendering,
  ED tab/gutter/syntax token table, CM message row geometry +
  composer specs.

## Implementation

Ported to Rust as STUMP #129:

- **`src/ui/widgets.rs`** — extended with new Wave-3 helpers:
  `draw_strip`, `draw_seg_separator`, `draw_conn_pill`, the
  `Tok` syntax-token enum + `tok_color`, and `draw_code_line`.
- **`src/ui/apps/filemanager.rs`** — rewritten. Header strip
  ("ENCRYPTED VAULT . AES-256-CTR + SHA-256 integrity" + file
  count). Column header (STATUS / FILENAME / SIZE / CIPHER /
  MERKLE OK). Real BatFS row iteration with [ENC]/[RAW] state
  badges, "f" prefix on names, KiB/MiB size formatting, cyan
  cipher label, green OK marker. Selection rendering: 1px
  cyan-dim inset border + 2px cyan underline at row bottom
  (visual style only — keyboard-arrow nav is a separate STUMP).
  Empty state: "(vault is empty - use 'write <name> <data>' in
  shell)" centered in dim text. Footer with file count, Merkle
  preview, hint.
- **`src/ui/apps/editor.rs`** — rewritten. 24px tab bar with 3
  tabs (kernel_main.rs active, lib.rs inactive, Cargo.toml dirty
  with amber dot) + cyan close glyphs + active underline + "+"
  new-tab slot. 56px gutter with right-aligned line numbers,
  current-line accent (1px cyan vertical strip + ink line number).
  Code area painted via `draw_code_line` over hardcoded
  `&[(Tok, &str)]` spans for the kernel_main.rs sample. Status
  strip: LANG RUST · ENC UTF-8 · POS Ln 17, Col 5 · LF UNIX +
  right-aligned amber READ ONLY badge.
- **`src/ui/apps/comms.rs`** — `render()` rewritten (connect /
  send / state machine preserved). 32px header with COMMS
  wordmark + connection pill (red DISCONNECTED / amber
  CONNECTING / green CONNECTED with peer value / red ERROR)
  + cipher pill + key pill (when connected). Timeline body
  with [HH:MM] timestamps, ">>" cyan / "<<" green direction
  arrows, "you"/"peer" sender labels, ink message text,
  bottom-anchored. 28px composer strip with cyan ">" prompt,
  ink typed text + cyan cursor block, char counter (dim <70 /
  amber 70-79 / red 80). Disconnected state shows centered
  "(no peer connected . use 'comms connect <ip>:<port>' in
  shell)" with disabled composer.

## Substitutions kept honest

| Design label | What we actually have |
|--------------|------------------------|
| Editor cursor blink | placeholder solid block (no input wired; READ ONLY) |
| ED tab close glyph | renders as `:x` since we don't have a true close icon |
| ED current-line bg tint | flat `#0E1F22` instead of true rgba alpha |
| FS row selection | rendered but always at row 0 (no keyboard nav yet) |
| FS file size unit | computed live from byte count (B/KiB/MiB), not preformatted |
| FS Merkle checkmark | "OK" text instead of `✓` (font is ASCII-only) |
| CM key exchange | session key derived SHA-256(peer_ip + ts), pill labeled "K" |

## Deviations from the design

1. **8×16 bitmap font** — same as prior waves. Spacing is
   chunkier than the JetBrains Mono mocks; layout still works.
2. **Editor syntax highlighting** is hardcoded line-by-line as
   `&[(Tok, &str)]` slices, not a real Rust tokenizer. When
   actual editing lands in a future STUMP, the tokenizer
   becomes a real pass over the buffer.
3. **No 1Hz blink anywhere** — composer cursor and editor
   cursor both render as solid blocks.
4. **Editor only renders one file's content** — tabs are
   visual. Switching tabs doesn't change the buffer (yet).
5. **Comms message rendering** uses a simple "show last N rows
   bottom-anchored" approach. No soft-wrapping or continuation
   rows yet — long messages just clip at the right edge.
