# Bat_OS Wave 2 — DS · NM · SK design source

JSX artifacts for the data-rich app trio (Dashboard, NetMon,
Security). Generated by Claude Design (April 2026) in the same
session as the lock-screen and desktop-shell so the visual
language carries forward.

## Files

- **`apps-ds-nm-sk.jsx`** — three pane components:
  - `DashboardPane` — SYSTEM (2×2 tiles) + SECURITY (KV rows) +
    full-width ARCHITECTURE panel with bat decoration.
  - `NetMonPane` — INTERFACE + FIREWALL (KV rows) + full-width
    SECURITY STACK flow diagram.
  - `SecurityPane` — full-width ACTIVE BATCAVES table + bottom-
    row SECURITY PIPELINE (KV) + INTEGRITY (Merkle row + KVs +
    audit mini-strip).
- **`apps-specs.jsx`** — implementation spec sheet. Defines the
  shared panel chrome, three body row types (KV / Tile / Caves
  list), status-dot palette mapping, and per-app layout.

## Implementation

Ported to Rust as STUMP #125:

- **`src/ui/widgets.rs`** — new shared widgets module. Hosts
  `draw_panel`, `draw_kv_row`, `draw_tile`, `draw_caves_*`,
  `draw_flow_box`, `draw_flow_arrow`, `draw_audit_strip`, and
  the `State` enum + `state_value_color` / `state_dot_colors`
  helpers. All three apps (and the rest, when they get
  redesigned) compose from here so panel styling / typography /
  status colors stay consistent.

- **`src/ui/apps/dashboard.rs`** — rewritten. SYSTEM tiles read
  uptime from `cntpct_el0`, free memory from `kernel::mm::frame::stats`,
  audit count from `security::audit::count`, network status from
  `virtio::net::is_ready`. SECURITY rows are mostly static labels
  matching what the kernel actually does (AES-256-CTR / SHA-256 /
  KDF 16 rounds / DENY ALL firewall / passphrase auth / caps
  enforced / live audit count).

- **`src/ui/apps/netmon.rs`** — rewritten. INTERFACE pulls real
  MAC + IP + GW. FIREWALL shows live allowed/blocked counts.
  SECURITY STACK draws the 6-box flow diagram (APP → TLS 1.3 →
  PIN VRFY → SOP → FIREWALL → WIRE) with cyan-dim arrows + cyan
  triangle heads. Narrow split-pane variant collapses to a 3×2
  grid with no arrows.

- **`src/ui/apps/security.rs`** — rewritten. ACTIVE BATCAVES
  iterates `cave::list`, decodes capability flags into the
  NET/RAW/DSP/FS pill set. SECURITY PIPELINE reads TLS mode
  live. INTEGRITY pulls the BatFS Merkle root, formats first 8
  bytes as 4-group hex, and ends with a 4-line audit mini-strip
  that mirrors the lock-screen boot log.

## Status mapping

| State    | Color | Use |
|----------|-------|-----|
| Ok       | green #22C55E | ACTIVE / READY / VERIFIED / live OK |
| Warn     | amber #F59E0B | STANDBY / WARN / RESEARCH mode |
| Fail     | red #EF4444   | DENIED / OFFLINE / FAIL |
| Plan     | dim #4B5563   | NOT WIRED / IDLE / pre-implementation |
| Neutral  | cyan #22D3EE  | live values: IPs, modes, hashes, counters |

## Substitutions kept honest (not faked)

| Design label | What we actually have |
|--------------|------------------------|
| YubiKey | Dropped — AUTH row says "PASSPHRASE" only |
| Argon2id | KDF row reads "16 ROUNDS · pre-Argon2id" (state=plan) |
| VPN connected | "STANDBY" (state=plan, dim) — stack present, not wired |
| MTU 1500 | dim (state=plan) — not actively negotiated |
| UTC clock | replaced with monotonic uptime since boot |
| OPEN PORTS = 0 | real value; "invisible-by-design" framing kept |

## Deviations from the design

1. **8×16 bitmap font, not JetBrains Mono.** Same caveat as
   prior waves. Tile numbers use `font::draw_str_scaled` at
   scale=2 (16×32) for the prominent number, smaller font for
   the label and unit.
2. **No 1Hz blink anywhere.** Lockscreen / status bar / shell
   prompt all stay static — periodic-tick hook is still a
   future STUMP.
3. **Flow-arrow triangle head** is a stack of 6 horizontal
   1px-tall fills approximating a triangle, not an actual
   anti-aliased arrow.
4. **Architecture-panel decorative bat** is the 36×24 simplified
   glyph (same as the title-bar one), not a 72×48 special raster
   — keeps the tile inventory at two sizes.
