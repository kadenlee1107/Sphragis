# Sphragis Lock Screen — design source

These JSX files are the design source-of-truth for the Sphragis auth
gate, generated by Claude Design (April 2026) from a prompt
describing this OS's security-first / operator-tactical aesthetic.

## Files

- **`bat-glyph.jsx`** — geometric bat with proper wing anatomy,
  finger-bone lines, and circuit-board nodes. 120×72 viewBox.
- **`lock-screen.jsx`** — full lock-screen renderer accepting a
  `state` prop (`"idle" | "typing" | "denied"`).
- **`specs.jsx`** — implementation spec sheet. Concrete hex values,
  font sizes, geometry on an 8px grid, raster notes for the paint
  stack.

## Implementation

Ported to Rust in `src/security/boot_screen.rs` as STUMP #116.
Color palette is a one-to-one mapping; geometry follows the spec
sheet; the geometric bat is rasterized via a tiny scanline
polygon-fill in the boot_screen module.

## Deviations from the design

What we ship differs from the source in three ways, deliberately:

1. **Aspirational labels swapped for real subsystem state.** The
   design said "Argon2id · 64MB · t=3", "TPM SEALED",
   "AES-XTS-512", "SECURE BOOT OK" — none of which we have. The
   implementation uses our actual values: SHA-256 KDF (16 rounds),
   BatFS MOUNTED, AES-256-CTR, M1N1 CHAINLOAD. When we land Argon2id
   / TPM / etc. we'll re-tighten the labels.

2. **8×16 bitmap font instead of JetBrains Mono.** `ui/truetype.rs`
   exists but isn't wired into the WM yet — bringing it in is a
   separate STUMP. The spacing / weight is chunkier than the mock;
   the layout still works.

3. **No 1Hz cursor blink, no scanline overlay.** Both deferred —
   blink needs a periodic-tick hook, the scanline overlay is cheap
   per pixel but ~340k pixel writes per repaint and we want to know
   it's wanted before paying for it.

## Updating the design

If we re-spin this in Claude Design, drop the new artifacts here,
diff against the current set, and update the Rust port. Keep the
README's "deviations" section in sync with what's actually shipped.
