# Sphragis Desktop Chrome + Shell — design source

These JSX files are the design source-of-truth for the post-auth
desktop chrome (title bar, content slot, status bar) and the SH
(shell) pane content. Generated by Claude Design (April 2026) in
the same session as the lock-screen artifacts so the visual
language carries forward.

## Files

- **`bat-mini.jsx`** — simplified 18×12 bat glyph for the title bar.
  Membrane silhouette only — no finger bones, eye slits, or circuit
  nodes (would collapse to noise at this raster).
- **`desktop-shell.jsx`** — the full desktop chrome: TitleBar (24px
  with brand block, 9 tabs, cave indicator), StatusBar (28px with
  ENCRYPTED · NET · TLS · JS · AUDIT segments + uptime), and three
  shell-pane states (empty, activity, mid-typing).
- **`shell-specs.jsx`** — implementation spec sheet. Concrete pixel
  positions, segment widths, color rules per shell-output category,
  and a "live state vs decoration" callout for status-bar segments.

## Implementation

Ported to Rust in:
- `src/ui/draw.rs` — shared paint primitives (scanline polygon-fill,
  Bresenham line, draw_border) and the two bat-glyph variants.
  Both `boot_screen.rs` and `wm.rs` use this module.
- `src/ui/wm.rs` — `draw_frame()` rewritten for the new title bar +
  status bar. `draw_status_bar()` reads live state from the TLS,
  JS, audit, and net subsystems — none of the segments are decoration
  except ENCRYPTED (which is always green post-boot by design).
- `src/ui/console.rs` — prompt typography updated: "sphragis" in INK,
  " > " in CYAN.
- `src/ui/desktop.rs` — `shell_banner()` replaced with a
  bat-glyph + structured wordmark/hints layout per the spec.

## Live status segments

| Segment | Source | Notes |
|---------|--------|-------|
| ENCRYPTED | static green dot | decoration; means "BatFS mounted" |
| NET | `net::ip::our_ip()` / `virtio::net::is_ready()` | "OFFLINE" in red if NIC down |
| TLS | `net::tls_pinning::current_mode()` | LOCKDOWN cyan / RESEARCH amber / OPEN red |
| JS | `browser::js::is_enabled()` | ON amber / OFF ink |
| AUDIT | `security::audit::count()` | shown as "N / 1024" |
| UPTIME | `cntpct_el0` / `cntfrq_el0` | format "Nd HH:MM:SS" |

## Deviations from the design

1. **Bitmap font, not JetBrains Mono.** Same caveat as the lock
   screen — `ui/truetype.rs` exists but isn't wired into the WM
   yet. Spacing is chunkier than the mock; layout still works.
2. **No 1Hz colon blink in the uptime segment.** Needs a periodic-
   tick hook the WM doesn't have yet.
3. **Output color rules per shell category** (audit indices in
   cyan, "OK" markers in green, etc.) aren't fully wired — most
   shell commands still call `console::puts(s)` with a single
   color. Wiring per-category color is a follow-up STUMP that
   touches every command's print path.
4. **No close button in the title bar.** The design didn't show
   one; pane close stays bound to Ctrl+Q. The existing close-button
   focus state machine in `wm.rs` is preserved (Tab cycling still
   works) but the visual indicator is gone since the new design
   doesn't have a place for it.
