## Recommended deletion plan

Two paths, each safe to take after a `git status` confirms a clean tree (so we can revert via `git restore` if anything goes wrong):

**Option A — full nuke** (recommended). One commit, remove the entire
`captures/` tree, add a tombstone note in `docs/`.

```sh
git rm -r captures/
echo 'captures/' >> .gitignore  # if not already
git add docs/CAPTURES_AUDIT.md   # this report stays
git commit -m 'cleanup: remove browser-era captures/ — see docs/CAPTURES_AUDIT.md'
```

**Option B — selective archive**. Keep one prefix group as a representative sample (recommended: the smallest one), nuke the rest.

```sh
# Pick a small bucket to keep as a sample (say `A`, ~149 files):
find captures/ -type f -not -name 'A*.png' -delete
git add -u
git commit -m 'cleanup: archive captures/ to A* sample only'
```

Either way, the full per-file inventory at `docs/captures_audit_full.tsv` survives the cleanup so the deleted names are still searchable.
